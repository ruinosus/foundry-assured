How it works

In DNA, Tenant is a first-class kernel dimension, orthogonal to layers. You build the kernel with a tenant bound to it and ask for the same agent — the source resolves the tenant's overlay and unions it over the platform base, shadowing by kind + name.

# base (shared — shipped once, owned by the platform) apps/backend/.dna/helpdesk/souls/concierge/SOUL.md # overlay (per tenant — the only file ACME writes) apps/backend/.dna/tenants/acme/scopes/helpdesk/souls/concierge/SOUL.md # composition Kernel(tenant="acme").instance("helpdesk").build_prompt("concierge-grounded")

No tenant → the base persona. tenant="acme" → ACME's persona, same shared body. A second tenant is a second folder — the base agent, its guardrail, and its eval-cases are never copied.
